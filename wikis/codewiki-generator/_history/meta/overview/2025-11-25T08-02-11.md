# CodeWiki Generator Overview

An intelligent documentation automation system that analyzes software repositories and generates comprehensive wiki-style documentation using Large Language Model (LLM) agents.

## Core Philosophy

**Code tells you what. Documentation tells you why. History tells you how.**

CodeWiki Generator automatically analyzes your codebase, discovers architectural patterns, and produces structured markdown documentation organized into concepts, components, and guides. The system practices **wiki-driven development**â€”building features, documenting them through this system, and validating quality through self-documentation.

## Key Capabilities

- **AI-Powered Analysis**: Uses Claude Sonnet 4.5 to analyze code and generate clear, insightful documentation
- **Specialized Agent System**: Dedicated agents for code analysis, documentation writing, architecture synthesis, and guide generation
- **Automatic Cross-Linking**: Discovers mentions and injects hyperlinks for seamless navigation
- **Category-Based Organization**: Documents organized into concepts/, components/, and guides/
- **Repository Fingerprinting**: Analyzes repository structure to guide documentation generation
- **Resilient JSON Parsing**: Progressive repair for handling unreliable LLM outputs
- **Self-Validating**: Successfully documents its own architecture at 87% quality
- **Comprehensive Testing**: 220+ passing tests with no API costs (all mocks)
- **MCP Integration**: Claude Code and other AI tools can query generated documentation via Model Context Protocol

### Core Components

- **Processor** - Orchestrates analysis and documentation generation
- **ArchitectureOverviewAgent** - Synthesizes high-level architectural insights
- **CodeAnalysisAgent** - Analyzes code structure and design patterns
- **DocumentationWriterAgent** - Generates markdown with code examples
- **GuideGenerationAgent** - Creates operational guides
- **LinkDiscoveryAgent** - Discovers and injects cross-page hyperlinks
- **WikiManager** - Handles markdown file operations and persistence
- **WikiIndexAgent** - Generates navigation structure

## MCP Server Integration

The MCP (Model Context Protocol) server enables AI assistants like Claude Code to query your generated wiki documentation for context while developing.

### Available Tools

1. **query_wiki**: Search documentation for relevant context
   - Intelligent context gathering based on task descriptions
   - Returns relevant pages with summaries and navigation links
   - Example: "How do I implement authentication?"

2. **request_documentation**: Queue documentation requests
   - Track missing documentation and priorities
   - Generate metrics for documentation gaps
   - Inform future improvement cycles

### Starting the Server

```bash
npm run mcp-server
# Or with custom wiki path:
node mcp-server.js --wiki ./wikis/your-project
```

Configure Claude Code to connect:

```json
{
  "mcpServers": {
    "codewiki": {
      "command": "node",
      "args": ["/path/to/CodeWiki-Generator/mcp-server.js"],
      "cwd": "/path/to/CodeWiki-Generator"
    }
  }
}
```

## How It Works: The Pipeline

1. **Repository Analysis** - Fingerprints structure and patterns
2. **Agent Dispatch** - Routes analysis to specialized LLM agents
3. **Content Generation** - Each agent generates documentation for its domain
4. **Quality Assurance** - Progressive JSON repair and validation
5. **Assembly** - Organizes content into category-based structure
6. **Cross-Linking** - Discovers mentions and injects hyperlinks
7. **Index Generation** - Creates navigable table of contents
